Dynamic Programming

A method of solving a complex problem by breaking it down into a collection of simpler subproblems, solving each of this subproblems just once, and storing their solutions.

It works with Overlapping Subproblems and Optimal Structure

Overlapping subproblems

A problem is said to have overlapping subproblems if it can broken down into subproblems which are reused several times

For example

Fibonacci Sequence

Every number after the first two is the sum of the two preceding once

Optimal substructure

A problem is said to have optimal substructure if an optimal solution can be constructed from optimal solutions of its subproblems

Enter Dynamic Programming

Using past knowledge to make solving a future problem easer

Enter Memoization

Sorting the results of expensive function call and returning the cached results when the same inputs occurs again
